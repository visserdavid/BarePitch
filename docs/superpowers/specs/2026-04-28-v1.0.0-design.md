# v1.0.0 Production-Ready Milestone — Design Spec

## Context

BarePitch is a plain PHP/MySQL soccer team management app. All core features are complete through v0.8.0 (auth, teams, players, matches, attendance, responsive layout, security hardening). Empty states are already implemented in all list views (views, CSS, i18n in EN and NL).

v1.0.0 is a milestone marker — not a live deployment. It declares the app functionally complete for its current scope and ready to serve as a baseline for future feature work.

---

## Goals

1. Commit `docs/sketches/` — preliminary research documents for the post-v1.0.0 roadmap
2. Add custom error pages (404 and 500) as standalone HTML files
3. Register the error pages in `public/.htaccess`
4. Tag and push the milestone

---

## What is NOT in scope

- Empty states: already fully implemented (views, CSS, i18n in EN/NL)
- Any features from the research sketches (clubs, seasons, training, lineup, match events, statistics)
- Account/profile page, password change, dashboard stats

---

## Section 1: Preliminary research documents

`docs/sketches/` is untracked. Commit it as-is with the prefix `docs:` before starting the error page work. No changes to the files themselves.

---

## Section 2: Error pages

### Files

- `public/404.html` — Page not found
- `public/500.html` — Something went wrong

### Design constraints

- **Standalone HTML** — no PHP, no bootstrap chain dependency. Works if PHP or the DB is down.
- **Link to existing static CSS** — `/assets/css/base.css`, `layout.css`, `components.css` so pages look like the app.
- **English only** — static HTML cannot use the `__()` i18n system. Acceptable trade-off for error pages.
- **No server-side logic** — no session, no CSRF, no auth.

### Structure

Both pages mirror the visual output of the existing header/footer but as inlined HTML:

```
<header class="site-header">
  <div class="container">
    <span class="site-name">BarePitch</span>
  </div>
</header>
<main class="container">
  <section>
    <h1>[Error heading]</h1>
    <p>[Short message]</p>
    <a href="/dashboard.php" class="btn btn-primary">Go to dashboard</a>
  </section>
</main>
```

### Copy

| Page | `<title>` | `<h1>` | `<p>` |
|---|---|---|---|
| 404.html | `404 — BarePitch` | `Page not found` | `The page you're looking for doesn't exist.` |
| 500.html | `500 — BarePitch` | `Something went wrong` | `An unexpected error occurred. Please try again.` |

---

## Section 3: .htaccess additions

Two lines added to the bottom of `public/.htaccess`:

```apache
ErrorDocument 404 /404.html
ErrorDocument 500 /500.html
```

These take effect under Apache. The PHP built-in dev server (`php -S`) ignores `.htaccess` — expected behavior.
