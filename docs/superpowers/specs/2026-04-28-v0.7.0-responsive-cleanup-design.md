# v0.7.0 Responsive Cleanup — Design Spec

## Context

BarePitch is a plain PHP/MySQL soccer team management app. The primary user is a coach on a mobile phone. As of v0.6.0 the CSS has zero media queries — nothing is responsive. This milestone fixes that without touching the desktop experience.

---

## Goals

- Make every page usable on a 390px-wide phone screen
- Add a bottom navigation bar for thumb-friendly navigation
- Keep desktop layout exactly as-is
- No new dependencies, no build tools

---

## Breakpoints & file structure

Single breakpoint: `640px`.

- Below 640px = mobile
- Above 640px = current desktop layout, unchanged

All responsive rules go in one new file: `public/assets/css/responsive.css`, loaded last in `layouts/header.php`. No changes to `base.css`, `layout.css`, `components.css`, or `forms.css`.

---

## Bottom navigation bar

A fixed 56px bar at the bottom of the screen, visible only on mobile (< 640px).

**Links:** Dashboard · Teams

**HTML:** Lives in `layouts/footer.php` so it appears on every page automatically.

**Context-sensitive "+ New" button:** List pages (teams, players, matches) pass a `$bottomNavNew` variable — an array with `url` and `label` — to render a "+ New" shortcut in the bottom nav. Pages that don't set it show only Dashboard and Teams.

**Body padding:** `body` gets `padding-bottom: 56px` on mobile so content is never hidden behind the bar.

**Styling:** Same neutral background as the site header (`#f4f4f4`, `border-top: 1px solid #ddd`). Three equal-width flex cells.

## Table responsive strategy

### Priority columns (visible on mobile)

| Table   | Keep on mobile       | Hide on mobile                                    |
|---------|----------------------|---------------------------------------------------|
| Teams   | Name                 | Season, Players count, Matches count, Status, Actions |
| Players | Shirt #, Name        | Status, Actions                                   |
| Matches | Date, Opponent       | Kickoff, Home/Away, Location, Status, Actions     |

Hidden columns use a `.col-hide-mobile` utility class on `<th>` and `<td>`, controlled in `responsive.css`.

### Tappable rows (replaces action buttons on mobile)

On mobile, action buttons are hidden and the entire table row becomes a tap target. Each `<tr>` receives a `data-href="..."` attribute in the view pointing to the primary action page (edit or detail):

- Teams → `/team_edit.php?id=X`
- Players → `/player_edit.php?id=X&team_id=Y`
- Matches → `/match.php?id=X&team_id=Y`

A small JS snippet in `layouts/footer.php` wires up the tap handler: reads `data-href` from each `<tr>` and calls `window.location` on click. The snippet wires all rows unconditionally — on desktop the buttons are still visible and take precedence; the row tap is a bonus affordance.

Any `<a>` link inside the primary name/opponent cell gets `pointer-events: none` on mobile so it doesn't intercept the row tap.

### Match detail page

Already mobile-friendly: the key-value table is capped at `max-width: 480px` and the attendance section uses flexbox rows. No changes needed.

---

## Page-level layout adjustments

### Page header rows

All list pages use an inline `style="display:flex;align-items:center;justify-content:space-between;margin-bottom:1.25rem;"` pattern. These inline styles are replaced with a `.page-header` CSS class. On mobile, `.page-header` stacks to `flex-direction: column; align-items: flex-start; gap: 0.75rem`.

Affected views: `teams/index.php`, `players/index.php`, `matches/index.php`.

### Form actions

`.form-actions` already uses flex. On mobile, add `flex-wrap: wrap` so buttons don't overflow on narrow screens.

## Out of scope

- No changes to auth pages (login is already narrow enough)
- No changes to form pages (create/edit) — inputs are already `width: 100%`
- No icon system — text-only nav bar
- No hamburger menu or off-canvas nav
